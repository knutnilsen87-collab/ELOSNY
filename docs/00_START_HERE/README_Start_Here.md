# Start Here

Use this folder before filling the rest of the package.

## Minimum first steps

1. Fill `Project_Profile_Card.md`
2. Fill `Project_Intake_and_Scoring.md`
3. Fill `Project_Type_Selection.md`
4. Decide which optional modules are active
5. Fill `Document_Priority_and_Source_of_Truth.md`
6. Confirm design source of truth: `docs/03_Design/UI_UX_Product_Spec.md`
7. Fill or initialize `docs/09_Project_Control/Status_Bundle_Contract.md` before execution or handoff

## Core principle

Do not write every document upfront.
Write the smallest set of documents that removes ambiguity for the current stage.

## Mandatory execution guardrail

Before any implementation work starts, create or update a `status_bundle`. Treat it as the operational truth for current state, evidence, risk, verification, next action, fallback, and repo-health status.

## Design authority

For Evida, use `docs/03_Design/UI_UX_Product_Spec.md` as the canonical UI/UX source of truth. Use `docs/03_Design/03_Evida_MVP_Figma_Flow.svg` as the visual MVP flow reference.
