# Definition of Done

A work item is done only if:
- code is merged
- tests pass
- logging/diagnostics are adequate
- docs are updated if needed
- acceptance criteria are met
- no known critical regressions remain
- latest `status_bundle` is updated
- verification evidence is linked or summarized
- repo health is `improved`, `preserved`, or explicitly accepted as degraded risk
- unresolved ambiguity does not claim green success
